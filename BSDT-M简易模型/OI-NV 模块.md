  
👉 把「未平仓量 OI」转成“资金强度”，再做统计标准化，最后输出一个 **方向+强度信号（-1~+1）**

# 一、T列：OI_NV（名义OI）

T = S × B

👉 含义：

- S：OI（张数）
- B：价格

👉 本质：  
**把“合约数量”转成“资金规模”**

📌 举例：

- 10000 张 × 60000 = 6亿名义资金

👉 为什么这么做？  
因为：

- 单看 OI（张数）没有意义（价格不同价值不同）
- 必须统一到“资金口径”

👉 结论：  
👉 **OI_NV = 市场真实杠杆规模**


# 二、U列：ΔOI_NV%（变化率）

U = T2 / T1 - 1

👉 含义：  
**名义资金的增长速度**

📌 举例：

- 从 6亿 → 6.6亿 = +10%

👉 本质：  
👉 **市场杠杆在加速还是减少**


# 三、V列：μ20_OI（20期均值）

过去20期 ΔOI_NV% 的平均值

👉 含义：  
👉 **正常状态下的资金变化水平**

📌 本质：  
这是你的“基准状态”

---

# 四、W列：σ20_OI（20期标准差）

过去20期 ΔOI_NV% 的波动范围

👉 含义：  
👉 **资金变化的正常波动幅度**

📌 本质：  
这是“噪音范围”

---

# 五、X列：z_OI（核心）

z = (当前变化 - 均值) / 标准差

👉 这个是整个模型最关键的东西

👉 含义：  
👉 **当前变化有多“异常”**

📌 举例：

- z = 0 → 正常
- z = 2 → 异常
- z = 3 → 极端

👉 本质：  
👉 **你在判断：现在是不是在“堆杠杆/去杠杆异常状态”**

---

# 六、Y列：dir_OI（方向）

SIGN((价格变化) × (OI变化))

👉 拆开看：

- B2 - B1 → 价格方向
- U2 → OI变化

👉 组合逻辑：

|价格|OI|含义|
|---|---|---|
|↑|↑|多头加仓（看多）|
|↓|↑|空头加仓（看空）|
|↑|↓|空头被平（挤空）|
|↓|↓|多头被平（踩踏）|

👉 本质：  
👉 **判断是谁在主导（多 / 空 / 被动清算）**

---

# 七、F列：OI偏差（最终输出）

F = dir × (clip(z, -3~3) / 3)

👉 分两部分：

### 1️⃣ clip(z, -3~3)

👉 把极端值限制住（防爆）

### 2️⃣ /3

👉 归一化到 [-1, +1]

---

👉 最终结果：  
👉 **一个带方向的强度信号**

|值|含义|
|---|---|
|+1|强多（大量加杠杆做多）|
|+0.5|中等多|
|0|中性|
|-0.5|中等空|
|-1|强空|

---

# 🔥 核心总结（一句话）

这一整套 OI-NV 做的是：

👉 **把 OI → 资金 → 变化 → 异常 → 方向 → 压缩成一个信号**

即：

OI → 资金规模 → 增速 → 标准化 → 判断多空 → 输出强度

# 🔥 用交易语言翻译（你更容易理解）

👉 这个模块其实在回答 3 个问题：

### 1️⃣ 市场有没有在加杠杆？

→ 看 ΔOI_NV%

### 2️⃣ 加得多不多（异常吗）？

→ 看 z_OI

### 3️⃣ 是谁在加？

→ 看 dir_OI

---

👉 最终：  
👉 **F（OI偏差） = 当前市场杠杆行为的“方向+强度”**

# 🔥 最关键的一点（很多人会忽略）

👉 这个模型不是看“绝对OI”  
👉 而是看：

变化 + 相对异常（z-score）

这就是为什么它能捕捉：

- 明斯基时刻前的堆杠杆
- 清算前的加速阶段
- 扫盘前的极端状态
