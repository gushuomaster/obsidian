# 🔁 一、最常用流程（记住这 3 步就够）

以后你每次改了 Obsidian 笔记：

git add .  
git commit -m "写你这次改了什么"  
git push

就这么简单 ✅

---

# 🧠 二、每一步在干嘛（你必须理解）

## 1️⃣ `git add .`

👉 把“改动”放进暂存区

包括：

- 新文件
- 修改过的文件
- 删除的文件

---

## 2️⃣ `git commit`

👉 生成一个“版本快照”

git commit -m "新增：OI模型笔记"

建议你写清楚：

|类型|示例|
|---|---|
|新内容|新增：BTC周期分析|
|修改|修改：OI-NV公式解释|
|修正|fix：修复路径错误|

---

## 3️⃣ `git push`

👉 同步到 GitHub

---

# ⚡ 三、进阶（你会越来越常用）

## ✅ 1. 查看改了什么

git status

👉 最重要命令（建议你常用）

---

## ✅ 2. 查看具体改动

git diff

---

## ✅ 3. 只提交部分文件（推荐）

git add 文件名.md  
git commit -m "只提交某个文件"

👉 避免乱提交（非常重要）

---

# ⚠️ 四、你这个 Obsidian 场景的注意点（重点）

你现在是知识库 + Git同步，这里有坑：

---

## ❗1. 不要频繁 `git add .`

👉 会把垃圾一起带上（比如缓存）

建议：

git status 先看  
再 add

---

## ❗2. .obsidian 建议处理掉

否则会：

- 插件冲突
- 排序错乱
- UI混乱

---

## ❗3. 中文文件名没问题（你现在就是）

Git 完全支持 👍

---

# 🔄 五、完整真实工作流（你以后就这么干）

### 🟢 场景：你写完一篇笔记

git status  
git add 2026-04-02.md  
git commit -m "新增：市场流动性结构分析"  
git push

---

### 🟡 场景：你改了一堆

git status  
git add .  
git commit -m "更新：OI模块 + 补充案例"  
git push

---

### 🔴 场景：你不小心加错了

git reset

👉 撤回 add（不会删文件）

---

# 🧠 六、进阶认知（你现在这个阶段必须知道）

Git 本质是：

工作区 → 暂存区 → 仓库 → GitHub

你每次做的是：

改文件 → add → commit → push

---

# 🚀 七、给你一个“职业级习惯”（非常关键）

每次写完就 commit：

👉 不要堆很多再提交

原因：

- 可回溯
- 可复盘
- 不怕写坏

---
